# Architecture

## Project structure

```
PDF-FILLER/
│
├── app.ts                            # Entry point — Express setup, middleware, routes
│
├── src/
│   ├── routes/
│   │   ├── ine.routes.ts             # POST /api/ine/procesar
│   │   ├── monto.routes.ts           # POST /api/monto/consultar
│   │   ├── pdf.routes.ts             # POST /api/pdf/generar
│   │   └── index.ts                  # Barrel export
│   │
│   ├── services/
│   │   ├── ine.services.ts           # OCR logic, field extraction
│   │   ├── monto.services.ts         # Table lookup, loan calculation
│   │   └── rellenar_pdf.services.ts  # Field mapping, PDF assembly
│   │
│   ├── utils/
│   │   ├── rellenar_pdf.utils.ts     # Low-level pdf-lib field writer
│   │   ├── tabla_cargar.utils.ts     # Excel reader, table indexer on startup
│   │   └── numero_letras.utils.ts    # Number to Spanish words (no dependencies)
│   │
│   └── types/
│       ├── ine.types.ts              # IneData interface
│       ├── tabla.types.ts            # TablaDescuento interface
│       ├── datos_extra.types.ts      # DatosExtra interface
│       └── index.ts                  # Barrel export
│
├── public/
│   └── generador.html                # Frontend — 3-tab form (Client · Amount · Extras)
│
├── scripts/                          # Utility / maintenance scripts
├── contrato.pdf                      # PDF template (not committed)
├── *.xlsx                            # Discount tables (not committed)
└── spa.traineddata                   # Tesseract Spanish model (not committed)
```

---

## Data flow

```
User uploads INE photo
        │
        ▼
POST /api/ine/procesar
  → Tesseract OCR reads image
  → Extracts nombre1, nombre2, apellidoPaterno, apellidoMaterno,
    curp, fechaNac, domicilio, claveElector, etc.
  → Returns IneData JSON
        │
        ▼  (frontend auto-fills Client tab fields)

User inputs montoNeto + plazoQuincenas
        │
        ▼
POST /api/monto/consultar
  → tabla_cargar.utils looks up the pre-indexed Excel table
  → Returns descuentoQuincenal, montoPagare, tasaMensual, cat, etc.
        │
        ▼  (frontend shows loan results)

User fills Extras tab → clicks "Generar Contrato PDF"
        │
        ▼
POST /api/pdf/generar
  → rellenar_pdf.services maps IneData + TablaDescuento + DatosExtra
    to the PDF template's named form fields
  → numero_letras converts amounts to Spanish words
  → rellenar_pdf.utils writes all fields via pdf-lib
  → Returns PDF binary → browser downloads the file
```

---

## Key design decisions

**Express static serving** — The frontend HTML is served via `express.static('public')`. No separate frontend server or CORS issues — everything on `localhost:3000`.

**Excel as data source** — Discount tables come from an official `.xlsx` file. `tabla_cargar.utils.ts` reads and indexes it on server startup so lookups are O(1) at runtime.

**PDF field mapping** — The contract PDF has named form fields. The service maps TypeScript interface properties to those field names. Unknown fields are warned and skipped — no hard crash.

**Number to words** — Amount in words (e.g. `DIEZ MIL PESOS 00/100 M.N.`) is generated by a custom zero-dependency utility covering amounts up to millions.
